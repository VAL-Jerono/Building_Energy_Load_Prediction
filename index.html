Key Empirical Findings

What the data actually say about building energy in this dataset.

📐
Height is the dominant lever
Overall Height (X5) is the single strongest predictor of both Y1 and Y2. Single-storey buildings carry ~18-22 kWh/m² less heating load than equivalent double-storey designs.
r ≈ 0.89 with Y1 · VIF = 31.2
🪟
Glazing has diminishing returns
The first 10pp increment of glazing carries the largest per-unit heating penalty (~3.8 kWh/m²). The NLS model formalises this: power law exponent b3 < 1 confirms diminishing returns.
GAM partial: non-linear positive · b3 < 1 (NLS)
📦
Compactness has an optimal band
The GAM partial dependence for compactness is U-shaped. The low-load range is approximately 0.73-0.79. Both very elongated and very cube-like buildings carry higher loads.
Optimal: 0.73-0.79 · VIF = 105.5 (collinear)
🧭
Orientation: near-zero in simulation
Orientation has near-zero correlation with Y1 and Y2 in this balanced simulation. Lasso confirms this by shrinking all orientation coefficients toward zero. Real East African sites are different.
All orientation dummies p > 0.30 in GAM
🔗
Severe multicollinearity in geometry
X1, X2, X4 (compactness, surface area, roof area) are near-perfectly collinear by geometric constraint -- VIF exceeds 100 for all three. OLS predictions remain valid; individual coefficients do not.
VIF > 100 for X1, X2, X4 · X'X near-singular
⚡
Height interactions are significant
F-tests confirm significant Height x Glazing and Height x Compactness interactions. Double-storey buildings respond differently to glazing increases than single-storey buildings.
Height-stratified GAM RMSE 1.23 vs 1.12 (pooled)

CRISP-DM Methodology

This study follows the Cross-Industry Standard Process for Data Mining, documented phase by phase.

1
Business Understanding
Three nested questions: regression (predict kWh/m²), classification (high vs low efficiency), diagnostic (which features matter and how). Each maps onto a model family in DSA 8305.
2
Data Understanding
Bimodal Y1/Y2 distributions (Shapiro-Wilk p < 10^-20) explained by Height's two-value structure. Correlation matrix reveals X1-X2 Pearson r = -0.99. LOWESS plots show non-linear glazing and compactness effects. VIF confirms geometric multicollinearity.
3
Data Preparation
One-hot encoding of Orientation (avoiding dummy trap by dropping one reference). 80/20 train-test split stratified by load quantile. Standardisation applied only for Ridge, PCR, and GAM.
4
Modelling
Seven model families in motivational sequence: OLS (baseline) → Ridge (multicollinearity) → PCR (geometric solution) → Polynomial (non-linearity) → Splines (piecewise smooth) → GAM (semi-parametric optimal) → Classification (logistic + decision tree).
5
Evaluation
Held-out RMSE and R² for all regression models. Confusion matrix, precision-recall curve, and threshold optimisation for classification. Bootstrap prediction intervals (N=150) revealed 24% empirical coverage vs 95% nominal -- motivating the 15% engineering margin.
6
Deployment
GAM predictions embedded in this interactive tool. JS approximation of GAM partial dependence functions. Classification threshold 0.20 applied (reduces false negatives from 13 to 1 in test set).

Version 2 Improvements

Six analytical additions over the original submission.

Improvement 1
Height x Feature Interaction Models
Height-stratified GAMs fitted separately for single and double-storey buildings. Diverging partial dependence curves confirm genuine interaction effects for compactness and glazing.
Improvement 2
Bootstrapped Prediction Intervals
N=150 bootstrap resamples generate empirical prediction intervals. Coverage was 24% vs 95% nominal -- confirming these intervals are not calibrated. Engineering margin used instead.
Improvement 3
PCR Component Signal Diagnostic
New plot shows correlation between each principal component and Y1/Y2 alongside variance explained. Empirically proves PCR discards signal-bearing components -- explaining its underperformance vs Ridge.
Improvement 4
OLS Singular Eigenvalue Explanation
Dedicated markdown cell explains why OLS predictions remain valid despite the near-singular X'X warning, and why individual collinear coefficients cannot be interpreted.
Improvement 5
Precision-Recall Threshold Optimisation
Classification threshold adjusted from 0.50 to 0.20 to minimise false negatives (under-specified HVAC). Reduces false negatives from 13 to 1 on the test set, accepting some precision loss.
Improvement 6
Classification Label Correction
Original global-median split gave 97% accuracy using only Height. Corrected to within-group median split, requiring all eight features. Honest accuracy: 83% logistic, 91% decision tree.

About the Tool

🎯
Purpose
Translate GAM predictions from the ENB2012 study into actionable design decisions at concept stage -- before any structural commitments are made.
⚙️
How predictions work
The assessment uses a JS approximation of the GAM's partial dependence functions. A real production deployment would call a /api/predict endpoint. Directional predictions are accurate; exact values carry a ±15% margin.
🤖
The AI assistant
The chat assistant is powered by Groq (Llama 3.3 70B). It is grounded in the study's system prompt -- all seven model families, key coefficients, and methodological caveats. Bring your own Groq API key (free tier available).
⚠️
Limitations
ENB2012 uses simulated data from Ecotect. No occupant behaviour, thermal mass, natural ventilation, or site-specific climate. Apply engineering judgment for real sites. Orientation effects are suppressed by simulation design balance.
